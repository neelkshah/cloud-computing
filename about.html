This storage service supports 'get' and 'put' operations for key-value pairs.
A key-value pair is replicated at 3 nodes, with one node acting as coordinator.
Upon the execution of a 'put' operation, the stored key-value pair and the nodes where it is stored are displayed.
Upon the successful execution of a 'get' operation, the following are displayed:
* The retrieved key-value pair
* The nodes from where it was retrieved
* The coordinator-node and version number
An unsuccessful 'get' operation is indicated as such.
